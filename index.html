История

React был создан Джорданом Уолком, инженером-программистом в Facebook, который выпустил ранний прототип React под названием «FaxJS». На него повлияли XHP, HTML библиотека компонентов для PHP. Впервые он был размещен на Facebook Новостная лента в 2011 году и позже Instagram в 2012. Его исходный код был открыт в JSConf US в мае 2013 года.

React Native, что позволяет Android, iOS, и UWP о разработке с помощью React было объявлено на конференции Facebook React Conf в феврале 2015 года, а исходный код был открыт в марте 2015 года.

18 апреля 2017 года Facebook объявил React Fiber, новый основной алгоритм библиотеки React для сборки пользовательские интерфейсы. React Fiber должен был стать основой для любых будущих улучшений и развития функций библиотеки React.[нуждается в обновлении]

26 сентября 2017 года React 16.0 был выпущен для широкой публики.

16 февраля 2019 года React 16.8 был выпущен для широкой публики. Релиз представил React Hooks.

10 августа 2020 года команда React объявила о первом выпуске-кандидате для React v17.0, известном как первый крупный выпуск без серьезных изменений в API React для разработчиков.

Что такое React?

React это JavaScript библиотека, созданная Facebook.

React это инструмент для создания компонентов пользовательского интерфейса.

Как работает React?

React создает виртуальный DOM в памяти.

Вместо того чтобы непосредственно манипулировать DOM браузера, React создает виртуальный DOM в памяти, где он выполняет все необходимые манипуляции, прежде чем вносить изменения в DOM браузера.

Реагирует только на изменения, которые должны быть изменены!

React узнает, какие изменения были сделаны, и изменяет только то, что должно быть изменено.

Пример использования

Ниже приведён пример использования React в HTML с JSX и JavaScript.


<div id="myReactApp"></div>

<script type="text/babel"></script>
  class Greeter extends React.Component {
    render() {
      return <h1>{this.props.greeting}</h1>
    }
  }

  ReactDOM.render( <Greeter greeting="Hello World!" />‚ document.getElementById( 'myReactApp'))
</script>

Класс Greeter — это React-компонент, принимающий свойство greeting. Метод ReactDOM.render отрисовывает экземпляр класса (компонента) Greeter со свойством greeting равным "Hello World" и вставляет отрисованный компонент в DOM-элемент с идентификатором myReactApp как вложенный элемент.


<div id="myReactApp">
  <h1>Hello World!</h1>
</div>


Оригинал

Однонаправленная передача данных

Свойства передаются от родительских компонентов к дочерним. Компоненты получают свойства как множество неизменяемых (англ. immutable) значений, поэтому компонент не может напрямую изменять свойства, но может вызывать изменения через callback-функции. Такой механизм называют «свойства вниз, события наверх».

Виртуальный DOM

React использует виртуальный DOM (англ. virtual DOM). React создаёт кэш-структуру в памяти, что позволяет вычислять разницу между предыдущим и текущим состояниями интерфейса для оптимального обновления DOM браузера. Таким образом программист может работать со страницей, считая, что она обновляется вся, но библиотека самостоятельно решает, какие компоненты страницы необходимо обновить.

JSX

JavaScript XML (JSX) — это расширение синтаксиса JavaScript, которое позволяет использовать HTML-подобный синтаксис для описания структуры интерфейса. Как правило, компоненты написаны с использованием JSX, но также есть возможность использования обычного JavaScript. JSX напоминает другой язык, созданный в компании Фейсбук для расширения PHP, XHP .

React Hooks

Хуки позволяют использовать состояние и другие возможности React без написания классов. Построение пользовательских хуков позволяет помещать логику компонента в повторно используемые функции.
